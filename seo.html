Search Engine Optimization:

Search Engine Optimization is important to any business. The reason, when done properly it delivers clicks and website visits that are organic. Organic is great because it means at no cost to the ad budget.

Additionally, you can’t afford to let your competitor get better search engine results than you. To even be considered by a web user you need to be in the top 3. Google might as well not even have more than one page.

One of the best approaches to tackle the challenges of SEO is to position your company as the leading authority on your industry. Convince Google you're the expert and they will reward your for your efforts.

I work for a great organization named ECKART. ECKART is an effects pigment manufacturer. Vacuum Metallized Pigments are very important to ECKART. Therefore, we want to be #1 in Google’s results page… and we are.

This doesn’t happen on accident. It was planned.

Every company should have an SEO Checklist. Outline your plan of attack for every stage of content development.

SEO takes place in the code, content and even the images you use.

As you can see, the results speak for themselves.

Cracking the Code to SEO:

As we move forward, it is important to remember that Google reads 100% of the HTML. This means we can put keywords in places the user doesn’t see.

The Head

This is a very important part of any website.

This is where we will give a lot of commands such as language, CSS and JavaScript locaitons. It is where we communicate with browser a lot information about "how" we want it to interpret a lot of the content to come.

Also, this is where a few of our Meta Tags are located.

  • Title
  • Keyword
  • Description
  • Slug

Title

The Title tag is where we tell search engines the title of the page. Keywords should be contained in your title.

Keyword

This is where we specifically tell Google what keyword we are focusing on. Google will read this tag, then scan the html for that keyword.

Description

Descriptions are a short paragraph (one or two lines) and less than 25 words. The goal here is to define the keyword in a way that relates to the content. I always like to make it so the description matches the first paragraph of the content (directly under H1). Don’t forget to include the keyword in the definition.

The Slug

Slugs are part of the URL that comes after your domain name. You want to make sure this contains the keyword. Be sure to use “-“ to separate words. Google will not recognize anything else as a space. It is important to manually add these to the slug because it is likely your framework will default spaces to “_” and this won’t work.

Headers

Users will read out headers, but depending on the design the user might not know the difference between H1, H2, H3, etc. Google does know the difference.

Google will be looking to ensure the content on the page is organized. This starts with proper headers and in the proper order. Remember, there is only one H1 allowed per page.

Paragraphs

Google will be looking to make sure we have used our keyword at a reasonable saturation level throughout the page. This means saturations should be around 3%.

Google will know the difference between using the keyword in the headers and the paragraphs because of the paragraph tags in the html.

Images for Search Engine Optimization

How we use images will play an important role in our optimized content.

We want to ensure the keyword is used in the following way:

  • File Title (hyphens-for-spacing)
  • Image Title
  • Alternate Description
  • Caption

File Title

When naming the files be sure the keyword is included. We also want to make sure we don’t use spaces. If you are using a framework, it is likely the framework will replace spaces with “_” and Google won’t recognize this the same way.

Image Title

The Image title will only be visible in the HTML, the user won’t see this. This is strictly for Google and for the visually impaired. Your title should include the keyword you are striving for.

Alternate Description

The Alternate description tag is a very important and overlooked tag. The purpose of an alternate description tag is three part:

  • It explains the image in case there is a loading failure.
  • The visually impaired will have a computer that reads them the description.
  • It is important to remember, this tag should include the keyword.

Caption

This is the final place where we will sneak in a keyword. Here we give a brief detail or description for the web user.

It is important to format it in a way that is subtle, but separate. I personally prefer color gray and italic. This lets the user know there is something here to read, however does it in a way that doesn’t take attention away from your image. Google will reward you for these types of actions.
